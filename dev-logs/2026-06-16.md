### Vercel 部署 — 问题追踪

| 尝试 | 错误 | 原因 | 修复 |
|------|------|------|------|
| ① | 无 vercel.json | 项目是 monorepo，Vercel 找不到根目录的 package.json | 创建 `vercel.json` |
| ② | `tsc: command not found` | `typescript` 在 devDependencies，Vercel 的 NODE_ENV=production 跳过了 devDeps | `client/package.json` build 脚本去掉 `tsc -b`，改为只 `vite build` |
| ③ | `cd: client: No such file or directory` | Vercel 检测到 `framework: "vite"` 后自动切换工作目录到 `client/`，再 `cd client` 变成找 `client/client/` | 去掉 `cd client` 前缀，命令和路径改为相对 `client/` 的写法 |
| ④ | `vite: command not found` | `vite` 同样在 devDependencies，自动 install 仍然没安装 | `buildCommand` 的 `npm install` 加 `--production=false` 参数 |

### 关键发现

- Vercel 的 `framework: "vite"` 会自动扫描 monorepo 中的 Vite 项目并切换工作目录
- Vercel 自动 `npm install` 时 NODE_ENV=production，跳过所有 devDependencies
- `npm install --production=false` 理论上应强制安装 devDeps，待验证
- 本地 `npm run build` 完全正常，产物在 `client/dist/`
